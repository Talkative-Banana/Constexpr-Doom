# Constexpr-Doom

A wasm interpreter in c++ constexpr evaluation to run doom in compile time. The interpreter supports both compile and runtime time execution of game. A stripped down version of wad was used to keep the memory consumption as low as possible, although compile time execution still requires close to 100GB of memory, which was alloted by allocating swapspace, imagine having 100GBs of RAM! ;).
